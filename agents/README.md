# Bears Spec Kit Agent Roles

This directory holds canonical local subagent TOMLs for the Bears workflow-overlay plugin.

Every TOML must declare top-level `role_kind`, `execution_class`, and `primary_eligible`.
Catalog-backed files mirror `assets/catalog/platform-role-catalog.v1.json` `roles[]`.
Profile-backed files mirror `agent_profile_mappings[]`.

## Codex custom-agent registration

Agent TOML files in `agents/` are the canonical Bears governance source. Codex does not discover these TOMLs from this plugin root by policy claim.

Use `plugins/bears/scripts/agent_registration_sync.py check --target user --json` for read-only drift evidence. Use `plugins/bears/scripts/agent_registration_sync.py sync --target user` or `--target repo` only when explicit materialization is required.

The script writes managed copies to `~/.codex/agents` or `.codex/agents`. It does not overwrite unmanaged local files.

`agents/openai.yaml` is UX metadata only, not custom-agent TOML registration. No SessionStart hook is enabled for this sync.

## Validation tooling

- `plugins/bears/scripts/validate_overlay.py`
  - `validate` checks plugin manifest basics, agent TOMLs, `.specify/extensions.yml` advisory state, and active feature artifacts.
  - Duplicate detection for Speckit skill sources is available via `--check-duplicates` and `--skill-source` options.
  - Use `--strict-overlay-skills` to enforce "Bears-only overlay" skill directories in `skills/` (`speckit-*` core skills are disallowed).

- `plugins/bears/scripts/platform_roles.py` validates the canonical role catalog and routes target paths to required roles.
- Secret Factory role closeout must keep the full control chain: `platform_roles.py validate`, `route`, `audit`, and both role + Secret Factory tests.
