## Agentic enterprise workflow requirements

- The plugin must keep the agentic enterprise workflow in `assets/catalog/agentic-enterprise-workflow.v1.json` and the role-scoped principles in `assets/catalog/agentic-enterprise-constitution.v1.json`.
- The repo domains are exactly `platform`, `gitops`, `infra`, and `product_infra`.
- Each scope must have one repo domain, measurable output, owner lineage, timebox, token budget, research-first stage, and validation path.
- L1 must own task decomposition and task matrix writing; L2 must own governance review and must not own task splitting.
- The workflow must not spawn one subagent per task; spawn is allowed only from a validated assignment packet after L2 governance review.
- The decision log must record user facts, user directives, agent decisions, contradictions, and user-input requests without raw chat text.
- Hooks must stay compact and must not execute tests, broad search, network calls, raw logs, workspace scans, or secret reads.
- Local-commit-owned validation must run `python3 scripts/agentic_enterprise_workflow.py validate` for workflow-core changes.
